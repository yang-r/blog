¶hexo部署的注意事项

  • npm的版本在稳定版(具体指令百度即可
  • hexo命令必须在其目录下执行 cd <hexo目录>
  • 在创建SSH路径时一路回车即可
  • 仓库名称为：名称.github.io
  • _config 修改时，冒号后面必有一个空格

¶hexo 常用指令

 hexo s==hexo server(本地查看网址为`http://localhost:4000/` 
 hexo new "title"  创建文章
 hexo new draft "title" 草稿
 hexo new publish title 发布草稿，即文章
 hexo clean  清除缓存文件和已生成的静态文件，部署在git上时最好先clean
 hexo g -d  文件生成后立即部署网站
 hexo g==generate
 hexo d==hexo deploy
 hexo version
 hexo server --drafts

¶关于文章

  • “阅读全文” 按钮
    在文章中插入<!-- more -->即可实现”阅读全文“的按钮

  • 页内跳转
    感觉是超链接的使用
    跳转到标题
    1.使用markdown： [名称](#id)
    2.html： <a href= #id >名称

  • 添加图片
    找到站点配置_config.yml将其中的post_asset_folder: 值改为true
    然后在hexo目录下执行命令npm install hexo-asset-image --save
    安装完成后创建文章(article)会生成同名文件夹,再将图片放入其中
    在文章中引用图片:![](articleName/pictureName.jpg)
    ps: 名称及其后缀大小写保持一致

  • 主题自带样式标签
    -label 标签

{% label 类型@内容%}
{% label info@info%}
{% label primary@primary%}
{% label success@success%}
{% label default@default%}
{% label warning@warning%}
{% label danger@danger%}

-note 标签

<div class="note default"><p>default</p></div>
`<div class="note default"><p>default</p></div>`
